Tratamento de Exceções

O tratamento de exceções é um mecanismo em Python para lidar com erros que ocorrem durante a execução de um programa. Em vez de o programa travar, você pode "capturar" o erro e lidar com ele de forma controlada.

Conceito

Uma **exceção** é um evento que interrompe o fluxo normal de um programa. O tratamento de exceções envolve o uso de blocos `try`, `except`, `else` e `finally` para gerenciar esses eventos.

  • `try`: O bloco de código que pode gerar uma exceção.
  • `except`: O bloco de código que é executado se uma exceção específica (ou qualquer exceção) ocorrer no bloco `try`.
  • `else`: Opcional. O código neste bloco é executado se nenhuma exceção ocorrer no bloco `try`.
  • `finally`: Opcional. O código neste bloco é *sempre* executado, independentemente de uma exceção ter ocorrido ou não. Útil para limpeza de recursos.

Aplicação

Usados para:

  • Prevenir a falha do programa devido a entradas inválidas do usuário.
  • Lidar com arquivos que não existem ou estão corrompidos.
  • Gerenciar problemas de conexão de rede ou banco de dados.
  • Garantir que recursos (como arquivos abertos) sejam fechados corretamente.

Exemplo de Código

`try-except` básico


try:
    numero = int(input("Digite um número inteiro: "))
    resultado = 10 / numero
    print(f"O resultado é: {resultado}")
except ValueError:
    print("Isso não é um número inteiro válido!")
except ZeroDivisionError:
    print("Não é possível dividir por zero!")
except Exception as e: # Captura qualquer outra exceção
    print(f"Ocorreu um erro inesperado: {e}")

`try-except-else-finally`


try:
    with open("arquivo.txt", "r") as f:
        conteudo = f.read()
    print("Conteúdo do arquivo:", conteudo)
except FileNotFoundError:
    print("Erro: O arquivo 'arquivo.txt' não foi encontrado.")
except IOError:
    print("Erro de I/O: Não foi possível ler o arquivo.")
else:
    print("Leitura do arquivo bem-sucedida.")
finally:
    print("Tentativa de leitura do arquivo concluída.")
